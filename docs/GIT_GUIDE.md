# 初心者向け：Git/GitHub 使い方ガイド

今の状態（WebGPU統合完了）を「合流」させたので、今後の作業での基本的な流れを解説します。

---

## 1. 普段の「保存」の流れ（毎日やる基本）

コードを書いて、キリが良いところで以下の3つのステップを行います。

1. **`git add .`**
   - 「変更したファイルを、保存箱に入れる準備をする」という命令です。
2. **`git commit -m "何をしたかメッセージ"`**
   - 「準備したファイルを、メッセージ付きで自分のパソコンに記録する」という命令です。
3. **`git push origin main`**
   - 「パソコンの記録を、GitHub（ネット上の金庫）に送る」という命令です。

> [!TIP]
> **迷ったらこれだけ:**
> 作業の終わりに `add` -> `commit` -> `push` をセットでやれば、コードが消える心配はありません。

---

## 2. 実験用の「分岐点（ブランチ）」を作る

大きな機能を作るときは、本番（main）とは別の道を作ります。

1. **新しい道を作る**:
   ```bash
   git checkout -b ブランチ名
   ```
2. **コードを書いて、自分の方に保存**:
   ```bash
   git add .
   git commit -m "メッセージ"
   ```
3. **GitHubに送る**:
   ```bash
   git push origin ブランチ名
   ```

---

## 3. 実験した成果を本番（main）に合流させる

作った機能が完成したら、本番のコードに合体させます。

1. **メインブランチに移動**:
   ```bash
   git checkout main
   ```
2. **合体（マージ）させる**:
   ```bash
   git merge 実験したブランチ名
   ```
3. **GitHubの本番を更新**:
   ```bash
   git push origin main
   ```

---

## 4. 使い終わったブランチを掃除する（整理整頓）

合流が終わったら、古いブランチは消してスッキリさせましょう。

1. **自分のPC内のブランチを消す**:
   ```bash
   git branch -d ブランチ名
   ```
2. **GitHub上のブランチも消す**:
   ```bash
   git push origin --delete ブランチ名
   ```

---

## 5. 困った時に使うコマンド

- **今どの道にいるか確認**: `git branch`
- **GitHub側も含めて全部見る**: `git branch -a`
- **しおり（タグ）を付ける**: `git tag タグ名 -m "メッセージ"` -> `git push origin タグ名`
- **書きかけを全て捨てて元に戻す**: `git checkout .`
